# 1、mysql启动错误处理

查看 log： Mac: /usr/local/var/mysql/lizhendeMacBook-Pro.local.err 根据log针对性的进行调整，包治百病

# 2、Mysql Incorrect datetime value

问题描述：

低版本的mysql中，数据库转储sql文件。导入到高版本的mysql中报错。报错提示信息如下： Incorrect datetime value: ‘0000-00-00 00:00:00’ for column ‘GMT_CLEANUP’ at row 解决办法： 登录mysql后执行：

# 说明：sql_mode是设置mysql应该支持哪些sql语法，以及哪种数据验证检查。
set global sql_mode='STRICT_TRANS_TABLES,STRICT_ALL_TABLES,ERROR_FOR_DIVISION_BY_ZERO';

# 3、ERROR : (2006, 'MySQL server has gone away’)

修改max_allowed_packed参数来避免：

# 设置：
set global max_allowed_packet=1024*1024*160;

# 查询：
show global variables like 'max_allowed_packet';
